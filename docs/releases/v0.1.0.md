# v0.1.0

Release date: 2026-06-25

## Summary

`ui-motion-workflow` `v0.1.0` is the first public open-source release of the project.

This version establishes the project's core identity:

- a portable workflow for UI direction, motion integration, and browser validation
- a Codex-first reference implementation
- documented adapters for Claude Code, Cursor, and VS Code
- a provider model built around `react-bits`, `magicui`, and `motion-primitives`

## What Is Included

### Core Workflow

- generic architecture for sequencing direction, implementation, and validation
- portable multi-agent usage guidance
- provider positioning and role-based selection guidance

### Reference Implementation

- Codex orchestrator skill
- repo-scoped Codex plugin artifact
- repo-scoped marketplace entry

### Environment Adapters

- Claude Code adapter via `implementations/claude-code/CLAUDE.md`
- Cursor adapter via `implementations/cursor/ui-motion-workflow.mdc`
- VS Code adapter via `implementations/vscode/copilot-instructions.md`

### Examples And Guidance

- marketing hero provider selection example
- dashboard motion polish example
- existing component enhancement example
- contribution and release-preparation documentation

## Why This Release Matters

This release turns the project from a concept into a usable open-source foundation.

It gives teams a stable workflow contract without forcing every environment into the same packaging model.

## Intentional Limits In v0.1.0

This release does not try to solve everything yet.

Current non-goals:

- universal plugin packaging across all runtimes
- exhaustive UI pattern coverage
- deep automation beyond the current workflow contract
- private or Powehi-specific overlays in the public core

## Recommended Starting Points

- Start with `docs/architecture.md` for the workflow model.
- Start with `docs/multi-agent-usage.md` for portability across environments.
- Start with `docs/codex-usage.md` if Codex is the target runtime.
- Start with `implementations/` if you want a concrete environment adapter.

## Next Focus

After `v0.1.0`, the most natural next steps are:

- refine install and usage guidance per environment
- test the workflow against more real host projects
- add stronger public examples and release cadence discipline
