# Validation Checklist

## Visual Coherence

- Does the motion match the page tone?
- Does it feel integrated rather than pasted in?
- Is there a clear focal point?
- Is the surrounding layout calm enough to support the effect?

## Motion Quality

- Is the animation too strong for the context?
- Does it start and stop cleanly?
- Does it remain understandable during interaction?
- Does it distract from core content or CTA text?

## Readability

- Is text readable while the effect runs?
- Is contrast still acceptable during animated states?
- Does a background effect compete with foreground content?

## Interaction Comfort

- Does hover or pointer-follow behavior feel helpful rather than noisy?
- Are touch or dense UI contexts still usable?
- Does the effect make controls harder to target or interpret?

## Responsive Behavior

- Does the effect still look correct on smaller widths?
- Does it crowd content on mobile layouts?
- Does it create clipping, overflow, or awkward spacing?

## Final Judgment

- Keep it if it improves the experience
- tone it down if it is impressive but not useful
- replace it if it contradicts the chosen design direction
