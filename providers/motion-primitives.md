# Motion Primitives Provider

## Role

`motion-primitives` is the composable motion-pattern provider in the reference stack.

Use it when the structure already exists and the missing piece is the motion behavior.

## Best Fit

- local component enhancement
- hover states
- reveal patterns
- subtle entrances
- reusable motion behaviors across product UI

## Strengths

- composable
- lower-level control
- better for refinement than replacement

## Risks

- requires more implementation work
- less immediate than importing a ready-made animated block

## Prefer It When

- the host component is already the right component
- you want micro-motion, not spectacle
- the design direction calls for restraint

## Avoid It When

- the page needs a ready-made hero effect
- the user expects a strong visual signature fast
